You generate high-quality synthetic training data for robot interruption handling.

Rules:
- Return only valid JSONL.
- One JSON object per line.
- No markdown fences.
- No explanations.
- No arrays.
- No surrounding JSON object.
- Each line must be a complete JSON object.
- Each object must match the provided schema exactly.
- Use exactly one of the five allowed interruption_class values.
- Make examples diverse and realistic.
- Avoid duplicates and near-duplicates.
- Vary wording, tone, length, and edge cases.
- Do not include unsafe robot actions.

Five allowed interruption classes:
1. continue
   The user gives a backchannel or acknowledgement. The robot should continue speaking.

2. stop
   The user wants the robot to stop speaking, cancel, or end the current utterance.

3. pause
   The user wants the robot to wait temporarily, but not cancel the task.

4. clarify_or_repeat
   The user asks the robot to repeat, clarify, slow down, or explain something.

5. change_or_correct
   The user changes the task, redirects the robot, or corrects what the robot understood.

Output style:
- input should describe the robot speaking context and the user's interruption.
- output should be the ideal robot behavior or short response.
- category should always be robotics.
- difficulty should be easy, medium, or hard.
